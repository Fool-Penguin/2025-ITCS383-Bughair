# ITCS383 Project Phase 2: Team Task Split (5 Members)

This document outlines the distribution of responsibilities for **Phase 2 Part 2 (The Maintenance)** to ensure all features and deliverables (D1–D5) are completed by the **30 April 2026** deadline.

---

## 👥 Roles and Responsibilities

### 1. 📱 Lead Mobile Developer
**Primary Goal:** Successful delivery of the Native Android App (Feature 1).

*   **Mobile Development:** Port all web functionalities to a native Android application.
*   **Feature Implementation:** Implement the 2 new PO features on the mobile client.
*   **Repository Management:** Maintain the separate mobile GitHub repository and ensure it is linked in the main `README.md`.
*   **D4 Support:** Provide technical input for the mobile SLOs (Software Lifecycle Objects) in the impact analysis.

### 2. 🌐 Lead Web & Backend Developer
**Primary Goal:** Backend stability, Web updates, and Infrastructure (D1).

*   **Cloud Deployment:** Deploy the backend to a cloud service (Render, Vercel, etc.).
*   **CI/CD Pipeline:** Set up GitHub Actions for automated builds and tests.
*   **API Adaptation:** Update backend logic to support the Android client (e.g., JWT authentication vs. cookies).
*   **Feature Implementation:** Implement the 2 new PO features on the web version.

### 3. 🧪 QA & Code Quality Specialist
**Primary Goal:** Ensuring high quality and testing standards (D2).

*   **Unit Testing:** Write tests to achieve **>90% test coverage** on all new code.
*   **SonarQube Integration:** Set up SonarQube Cloud and monitor quality gates.
*   **Deliverable D2:** Create `D2_CODE_QUALITY.md`, comparing code quality reports from before and after maintenance.
*   **Verification:** Perform final inspections to ensure all 8 CRs meet their "Verification Method."

### 4. 🔍 Systems Analyst
**Primary Goal:** Maintenance documentation and impact mapping (D3, D4, D5).

*   **Deliverable D3:** Create `D3_CHANGE_REQUESTS.md` with 8 specific Change Request tables (2 of each type).
*   **Deliverable D4:** Create `D4_IMPACT_ANALYSIS.md`, including:
    *   Traceability Graphs (Full vs. Affected).
    *   Directed Graph of SLOs.
    *   Connectivity Matrix.
    *   Written analysis of maintenance difficulty.
*   **Deliverable D5:** Maintain the `D5_AI-USAGE.md` report.

### 📋 5. Project Manager & PO Liaison
**Primary Goal:** Coordination, Communication, and Presentation.

*   **External Liaison:** Contact the assigned Product Owner group to finalize requirements for Features 2 & 3.
*   **Internal Product Owner:** Define 2 reasonable and doable feature requests for your assigned "customer" group.
*   **Integration:** Ensure all deliverables (D1-D5) are formatted correctly and committed on time.
*   **Presentation Lead:** Organize the Live Demo and prepare justification for the maintenance decisions for the May 1st presentation.
